## SECTION 2 — PER-STAGE CARDS (as-ran)

### 2a. Light pass — `prompts/step1-light-pass.md` (533 lines)
Consolidated file holding **3 built agents + 1 adversarial cross-check + deterministic scaffold + full JSON schema + closed enums + hook spec**. README calls these 4 "agents": Finder / Light Brand Analysis / Light Space Analysis / Gap Analysis — but the file itself defines the built agents as Finder, Roster Verifier (1.5), Dumper, Space Classifier (the README's "Light Brand Analysis" = Dumper, "Light Space Analysis"+"Gap Analysis" map to Classifier + the downstream market-selection skill).

- **AGENT 1 — FINDER** (1 agent, product-start). Decision: which brands clear the KEEP BAR (real product+URL+plausible competitor) and NET rule (substitutability OR bet-similarity); RELEVANCE-DROP borderline, don't keep-and-flag. Floor: ≥12 varied queries per lane × 3 lanes. Inputs: bet brief (prose, layer A — never hook-validated), `product`/`category`. Outputs `brands.json` fields: `brand, slug, url, product_observed, sells_observed, channel (dtc|marketplace|crowdfunding), lane (major|crowdfunding|marketplace), ads_flag (yes|no|unsure), crowdfunding{platform,launch_date,goal,raised,pct_funded,status} | null, found_by[], relevance, price_points[], revenue_est{...}=null (script fills), demand_trend{shape,window,source,basis}` (fetch.js fills), plus `dropped[]`. DR knowledge: definitions.md referenced ("load definitions.md") — no DR-marketing bundle. Model/count: 1 agent.
- **AGENT 1.5 — ROSTER VERIFIER** (adversarial, runs after Finder, before fetch). Decisions: SLOP CHECK (each kept brand vs keep bar), GAP CHECK (missing competitors, web-verified), DEDUP/CHANNEL SANITY. Output JSON: `slop_flags[]{slug,reason,recommend:"drop|keep-but-bench"}, missing_brands[]{brand,url,why,lane}, dedup_channel_fixes[]{slug,issue,fix}, verdict:"clean|needs-human-review", summary`. Recommends, doesn't rewrite — human applies to `brands.json`. 1 agent.
- **AGENT 2 — DUMPER** (1/brand parallel, verbatim). Reads ONLY `corpus/<SLUG>/clean/` + `ads/<SLUG>.json`; never fetches; never classifies. Emits `dump.json`: `slug, creatives[]{creative_id, type:"ad|landing_page|product_page", url, ad_id, page_id, start_date, run_length_days, format, niche_raw, angle_raw, canonical_angle:null, angle_basis:"observed|inferred", pitches[]{claims[](verbatim), mechanism[], problem_um_raw[], transformation:null}, canonical_niche:null, linked_funnel_id, link_basis:"cta_url|inferred|unresolved", multi}, notes`. `claims[]` MUST be verbatim substrings of clean copy (hook-enforced). DR: definitions.md only.
- **AGENT 3 — SPACE CLASSIFIER** (1 agent, reads ALL dumps — only judgment stage). Decisions: cluster claims→canonical TRANSFORMATION per pitch; TYPE each claim (CLAIM_TYPE_ENUM `direct|enlarged|mechanism|enhanced`); cluster niches/angles/bet_types; build COMBOS (transformation×niche); per-brand bet_type call (OPEN, page-quoted basis) + demand_trend carry + sophistication stage (1–5, read off claim_type distribution); mechanisms-in-play shared(3+)/unique(≤2) ownability per cell + space-wide. Emits `space-map.json`: `transformations[], niches[], angles[], bet_types[], mechanisms_in_play[]{canonical,raw_variants,brand_count,ownability}, combos[]{transformation,niche,brand_count,creative_count,brands[],claim_count,enhanced_claim_count,claims[]{text,type},mechanisms_in_play[],anti_fluke{brands_at_scale,qualifying_creatives}}, per_brand[]{slug,transformations,niches,bet_type,bet_type_basis,demand_trend{shape,window,source,basis},sophistication}, saturation[]`. DR: definitions.md + `agents/implementation-notes.md` layer-discipline examples (inline, no DR-marketing bundle). 1 agent.
- **Determinism scaffold (scripts):** `fetch.js` (Playwright homepage+LP-hunt+Trends), `clean.js`, `adlib-one.js`, `dedupe.js`, `revenue-est.js`. **Hooks (PostToolUse on Write):** DUMPER / CLASSIFIER / REVENUE / FINDER validators (route.js dispatches by filename).
- **No "1.5 slop-check/missing-brands" as a separate stage beyond Roster Verifier** — that IS the slop+gap step.
- Closed enums: `CHANNEL_ENUM`, `CLAIM_TYPE_ENUM`, `DEMAND_TREND_SHAPE_ENUM (steady|rising|parabolic-spike|declining|unknown)`. Open (clustered): claims·mechanism·niche·angle·bet_type.

## SECTION 3 — ORCHESTRATION REALITY (workflow.md, CLAUDE.md, capability_inventory.md, run-retrospective.md + the run debug docs)

**How stages were actually launched.** `run-retrospective.md §7` states it plainly: *"Claude Code IS the orchestrator; stage prompts become `.claude/commands/*.md` slash commands; data sources become MCP servers (future). [NEW, architecture intent]"* and *"GSD ceremony was abandoned for this project — too heavy for learn-by-building."* In practice each skill is the orchestrator-in-the-main-loop that fans out subagents (Router, Section Analyzer, relevance-bucket, etc.) via the Task tool and runs deterministic scripts itself. `_marketing-decisions/INDEX.md` confirms `/funnel-architect`, `/market-selection`, `/funnel-deep-pass`, `/copywrite` are invoked as slash commands.

**`CLAUDE.md` locked design rules:** one job per agent, smallest split, route to right executor; deterministic jobs (fetch, clean, dedupe, count/score, store, validate) = scripts/hooks, only judgment jobs (query design, classify, extract, synthesize) = agents — *"An agent that cleans data is a category error."* Plus `no-overwrite-v1` versioning (D-07/08/09; a guard hook is explicitly DEFERRED).

**Order actually run on Arduview:** light pass (fetch.js across 20 brands → classifier → root `space-map.json`) → `/market-selection` (produces filtered `runs/arduview/space-map.json` with `_chosen_cell`, run_transformation=`novelty-object-own`, run_niche=`edc-aesthetic-collectors`) → `/funnel-deep-pass` over 4 funnels (Playdate, Pocket Operator, Divoom — DTC no-ads; GameShell — crowdfunding KS-404) → `/funnel-architect` (→ `FUNNEL-DESIGN.md` v2) → LP built (`runs/arduview/site/`: index.html, deposit.html, order-confirmation-email.html, styles.css, script.js, STYLE-LOCK.md) → asset-classify (25 records) → pipeline-audit. The chosen cell for the deep pass was `novelty-object-own × edc-aesthetic-collectors`.

**Human intervention points:** (1) operator authors `pre-research-plan.md` bet brief; (2) Roster Verifier flags → human applies to `brands.json` before fetch; (3) operator picks the NTP from market-selection survivors (D1); (4) operator pushes/revises the funnel architect conversationally; (5) asset human pick gate (brick 6); (6) all of `_marketing-decisions/*` are operator calls deferred to the human.

**Settings-hook reality** (`.claude/settings.json`): one `PostToolUse` matcher `Write` → `node tools/hooks/route.js "$CLAUDE_TOOL_INPUT_PATH" validate-finder validate-dumper validate-classifier validate-revenue`. route.js dispatches by filename: `brands.json → validate-finder + validate-revenue`, `dump.json → validate-dumper`, `space-map.json → validate-classifier`, else pass. **These settings hooks do NOT fire inside subagents** — the entire pipeline's "auto-injection" / validate-on-write story collapses for subagent stages, which is the #1 documented orchestration failure (below).
