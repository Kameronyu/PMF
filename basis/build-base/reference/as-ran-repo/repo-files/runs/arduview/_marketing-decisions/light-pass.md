# Light Pass — marketing decisions you own

> Surfaced from the run retros. The "what went wrong" is signal; the fixes are yours to decide.

## Gadget/maker spaces don't run pain-causal framing — mechanism IS the signal

**What happened:** The classifier treated `problem_um_raw` and `mechanism[]` as equivalent signal sources. For gadget/maker product pages, pain-causal "here's why you suffer" framing is absent by design — the data was empty not because it was missed, but because it doesn't exist on these pages.

**Why it matters (marketing):** If mechanism is the real signal for this product class and pain is structurally absent, then any copy framework that leads with pain is working from a category error. How you weight mechanism vs. problem language in your positioning is a judgment call that can't be read from competitor pages.

**Your call:** For a gadget/hardware product like Arduview, decide: is the offer mechanism-led or transformation-led? Which persuasion frame do you anchor the funnel on — what it does, or what owning it becomes?
