<objective>
Build the manifest contract: `asset-map-rank.js` (brick 5 — route claims to sections via the table, rank, emit gap_list), `asset-emit.js` (brick 7 — write the queryable JSON + the launch-format MD manifests), and `asset-upload.js` (brick 8 — backfill CDN URLs from a url-map). This turns the validated per-asset records into the artifact the LP-build/copywriter agents (Phase 15) consume.

Purpose: The whole point of the classifier is a manifest the builder queries by CLAIM, not slot. asset-map-rank.js keeps section-assignment in the deterministic editable table (auditable, no re-running vision) and emits the gap_list — sections with no strong+clean asset (the proof's no-clean-hero case, the thing that tells the operator what to reshoot). asset-emit.js writes both the machine-queryable `images.json`/`videos.json` (filter by claim+strong+clean, rank by min_safe_use) and the human-readable `IMAGES.md`/`VIDEOS.md` mirroring `launch/inkleaf-launch/` format. asset-upload.js handles the Shopify CDN gotcha: image URLs are constructable, video URLs are hashed and must be captured after upload — so it accepts a `url-map.json` that works with manual OR API upload (credentialed automation is out of scope, rides M2).

Output: `runs/<space>/asset-classify/images.json` + `videos.json` + `IMAGES.md` + `VIDEOS.md`, with a gap_list and (after url-map) CDN URLs.
</objective>

Routing logic (the brick-specific part — section routing stays in the table, NOT in code):
- For each record, for each `demonstrates[]` entry: look up `claim` in the section-table; add the record to that claim's `primary` and `also` sections. **Honor the `requires.display_state: on_legible` conditional** — `see_through_display_on` routes to `hero` ONLY when the record's `display_state == "on_legible"`; otherwise route it to its `also` sections only. - **Rank** candidates per section: sort by demonstration strength (strong > partial > incidental) then technical fit (`min_safe_use`: hero > section > thumb). Skip records where `dup_of` is set (deduped) for primary-pick ranking; skip records with non-empty `disqualifiers` for hero/primary slots (downgrade, not delete).
- **gap_list:** for each section in the section list, if NO candidate is `strong` AND clean (`disqualifiers: []` AND `dup_of: null`), add the section to `gap_list` with a reason (e.g. `"no strong+clean asset — reshoot or source"`). The Arduview UAT target: every shot has a busy/novelty background → `hero` lands in the gap_list (no clean-studio hero).

Output: write `runs/<space>/asset-classify/ranked.json` = `{ sections: { <section>: [ {id, role, claim, strength, min_safe_use} ... ranked ] }, gap_list: [...], _provenance }`. Also stamp `eligible_sections` onto each source record (asset-emit.js reads these). Handle images and videos: video records route via the same claim table but land in `hero_loop`/`feature_demo` slots (use `eligible_slots` + `best_use` for video routing). Use S2 sanitizer on `--space`; S4 log+summary+exit.
