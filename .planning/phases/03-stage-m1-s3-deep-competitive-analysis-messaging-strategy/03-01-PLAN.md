<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ad creative -> harvested destination_url -> Playwright render | The destination URL is attacker-influenceable data harvested from a competitor's ad creative. funnel-assemble.js navigates a real browser to it (SSRF surface). |
| external LP -> landing_page_body (raw) | Arbitrary untrusted HTML/JS from a competitor page is fetched, rendered, and stored raw. |
| Meta Ad Library -> adlib-one.js scrape | Third-party site scraping (rate-limit / ToS). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information disclosure (SSRF) | funnel-assemble.js `fetchPage(destination_url)` | mitigate | Before goto: parse the normalized URL; reject/skip (stamp `ambiguous_destination` + log) any URL whose host resolves to a private/loopback/link-local/metadata range (127.0.0.0/8, 10/8, 172.16/12, 192.168/16, 169.254.0.0/16 incl. 169.254.169.254 cloud metadata, ::1, fc00::/7) or non-http(s) scheme. Allow only http/https. |
| T-03-02 | Denial of service (resource exhaustion) | funnel-assemble.js Playwright render of untrusted LP | mitigate | Hard `timeout: 60000` on goto + `30000` on networkidle (already in fetchPage); cap "Read more" expansion clicks; per-funnel try/catch so one hostile/slow page never aborts the batch; run headless with `--no-sandbox` already in boilerplate — keep the page context disposable per funnel. |
| T-03-03 | Tampering / injection (deferred downstream) | landing_page_body stored raw -> later fed to Section Analyzer LLM | accept (mitigated in Plan 03) | Stored raw here BY DESIGN (cleaner/analyzer are separate bricks). The prompt-injection risk from competitor page copy is mitigated at the analyzer boundary in Plan 03 (treat scraped text as untrusted DATA, not instructions). No mitigation owed at this layer beyond not executing the content. |
| T-03-04 | Repudiation / ToS | adlib-one.js + Meta Ad Library scrape | accept | Public Ad Library data, point-in-time snapshot, low volume (1-2 brands at a time per D-17). Existing fingerprint-mask + single-advertiser pulls keep volume within informal limits; no credentials handled. |
</threat_model>
