## Directory Purposes

**`engine/bricks/`:**
- Purpose: All deterministic CLI scripts (fetch, clean, score, funnel, asset, aggregate, audit)
- Contains: Single-job executors per CLAUDE.md law (no agent logic, no LLM calls, pure mechanical transforms)
- Key files:
  - **Fetch pipeline:** `fetch.js` (homepage + LP paths + Google Trends), `crowdfund-fetch.js`, `adlib-one.js` / `adlib-sweep.js` (Meta ad library)
  - **Clean pipeline:** `clean.js` (HTML → markdown), `funnel-clean.js` (section markers), `dedupe.js` (brand dedup)
  - **Funnel pipeline:** `funnel-assemble.js` (URL normalize + cluster), `funnel-score.js` (two-currency validation), `funnel-store.js` (per-funnel JSON), `funnel-claim-tally.js` (saturation), `funnel-analyzer-context.js` (assembles context block for analyzer)
  - **Embedding/RAG:** `funnel-vectorize.js` (index builder), `funnel-rag-query.js` (semantic retrieval)
  - **Asset pipeline:** `asset-fetch.js` (glob local assets), `asset-map-rank.js` (route claims to sections), `asset-emit.js` (write manifests)
  - **Aggregate:** `revenue-est.js` (traffic formula), `aggregate-mechanisms-in-play.js` (saturation count)
  - **Audit:** `audit-inject.js`, `audit-resolve.js`, `validate-receipt.js`, `validate-strip.js`
  - **Shared lib:** `lib/embed.js` (Voyage/stub), `lib/adlib-graphql.js` (GraphQL parsing), `lib/trends-parse.js` (Trends data extraction)
  - **Asset Python:** `asset/probe.py` (EXIF+phash), `asset/probe_video.py` (video metadata), `asset/frame-grab.py`, `asset/sample_montage.py`, `asset/video-assemble.py` (EDL-based video)
  - **Config:** `asset/section-table.json` (section-list mapping for asset-map-rank.js)

**`engine/hooks/`:**
- Purpose: Firing layer — gate every agent output, bundle DR context, firewall marketing writes
- Contains: All validators, router, DR injectors, firewall
- Patterns:
  - **Router** (`route.js`): Basename dispatch (brands.json → finder+revenue, dump.json → dumper, space-map.json → classifier, *-beliefs.json → analyzer)
  - **Validators:** Exit 0 (pass) / 2 (reject); import `../contracts/enums.json`; enforce enum membership, required fields, verbatim traceability
  - **DR Injectors:** `inject-dr.js` (Section Analyzer), `inject-market-selection-dr.js`, `inject-funnel-architect-dr.js`, `inject-copywriter-dr.js` — read DR files via hardcoded allowlist + path-traversal guard; `--dr-dir` override
  - **Firewall:** `guard-marketing.js` reads `off-limits.json`, blocks Write/Edit/MultiEdit to operator-owned paths

**`engine/integrations/`:**
- Purpose: External service automation (Shopify, Surge, Cloudflare, Klavioy, Chrome CDP)
- Contains: Service-specific code + shared credential seam
- Patterns:
  - Each service folder contains its own files (no cross-folder imports except `lib-creds.js`)
  - All scripts accept `--creds=<path>` flag (or env var, or `__dirname` fallback — via `lib-creds.js`)
  - Shopify: Funnel HTML → Liquid templates + pages + asset upload
  - Surge: pty-driven `npx surge` (cli automation for static deploy)
  - Cloudflare: REST client + DNS reconciler
  - Klavioy: Minimal REST API (email)
  - CDP: Raw WebSocket driver + Playwright wrapper + real mouse clicks + Windows-only loopback forwarder

**`engine/contracts/`:**
- Purpose: SINGLE SOURCE OF TRUTH for all wiring definitions
- Contains: Capability registry, enums, schemas, naming law, reuse index, error notes, test suites, health summary
- Key files:
  - `REGISTRY.json/.md` — 26 capabilities (health, I/O, run, deps, error-notes); generated from brick header comments, hand-flipped for health
  - `enums.json` — CHANNEL, LANE, CLAIM_TYPE, DEMAND_TREND_SHAPE, EXECUTION_TYPE, PROOF_TIER, MOVE, BELIEF_ID_ANCHORS, BELIEF_KIND, AWARENESS_ENTRY, ROUTING_FLAG, SHOT_TYPE, DISQUALIFIER, ASSET_STRENGTH, BEST_USE, DISPLAY_STATE
  - `off-limits.json` — marketing deny-list (operator-owned paths the engine never edits)
  - `schemas/` — field shapes (brands.schema.json, belief-record.schema.json, funnel-fields.schema.json, asset-record.schema.json, space-map.schema.json, dump.schema.json)
  - `NAMING.md` — canonical naming law (file patterns: `<domain>-<verb>.js`, identifiers, place-by-job-not-origin)
  - `REUSE-INDEX.md` — what's proven wiring (fold IN) vs marketing-authored (off-limits, fold AT R5)
  - `ERROR-NOTES.md` — doctor patterns (known bugs + H-pass fixes: #funnel-score-input, #trends-0pct-fill, #funnel-clean-md-headings, #cred-seam, etc.)
  - `21-CLOSEOUT-SUMMARY.md` — P21 hardening summary (health flip to 22/26 working + 1 dry-run + 1 partial + 2 deferred)
  - `h6-*.sh` — smoke suites (h6-all.sh = 14/14 green; h6-clean.sh, h6-dedupe.sh, h6-funnel-assemble.sh, h6-claim-tally.sh, etc.)

**`engine/prompts/`:**

- Contains: `_specs/image-classifier-brick.md` (deterministic wiring for asset probe/map-rank/emit chain)

**`engine/_fixture/`:**

  - `funnels/sample.json` — sample funnel for h6-claim-tally.sh (golden comparison)

## Naming Conventions

**Files:**
- **S-brick (deterministic script):** `<domain>-<verb>.js` (verb ∈ {fetch, clean, dedupe, normalize, assemble, score, store, vectorize, query, tally, emit, rank, upload, estimate, probe, grab, deploy, drive, route, validate, inject, resolve})
  - Examples: `fetch.js`, `funnel-assemble.js`, `asset-map-rank.js`, `revenue-estimate.js`
- **Python brick:** `<domain>-<verb>.py` or `<noun>_<verb>.py` (preserve snake_case if existing)
  - Examples: `probe_video.py`, `frame-grab.py`, `video-assemble.py`, `surge_drive.py`
- **Integration:** `<service>-<verb>.{js,cjs,py}` under `integrations/<service>/`
  - Examples: `shopify-deploy-funnel.js`, `surge_drive.py`, `cdp.cjs`
- **Validator:** `validate-<producer>.js` (producer = agent/brick whose output it gates)
  - Examples: `validate-finder.js`, `validate-analyzer.js`, `validate-asset-record.js`
- **Injector:** `inject-<consumer>-dr.js` (consumer = agent the bundle feeds)
  - Examples: `inject-dr.js`, `inject-market-selection-dr.js`, `inject-copywriter-dr.js`
- **Contract:** enum keys `UPPER_SNAKE` in `enums.json`; record schema `<record>.schema.json`

**Directories:**
- **Logical grouping:** `bricks/`, `hooks/`, `integrations/`, `contracts/`, `prompts/`, `skills/`, `_fixture/`
- **Physical structure:** Flat within `bricks/`, `hooks/` (relocation-only, no sub-dirs for logical grouping — registry is the logical view)
- **External service:** `integrations/<service>/` (one dir per service; shared `lib-creds.js` at root)
- **Asset specialization:** `bricks/asset/` (Python pixel work) + `bricks/lib/` (shared JS libraries)

**Identifiers:**
- **Capability ID:** kebab-case, verb-or-noun-led, stable: `lp-clean-markdown`, `reddit-extract`, `shopify-deploy`, `product-video-assemble`, `embeddings-rag`
- **Agent label (PROVISIONAL):** `<step>-<job>`: `s1-finder`, `s2-section-analyzer`, `s6-asset-classify` (reconcile against operator's rebuilt I/O contracts)
- **Artifact path:** `runs/<space>/<artifact>` (per-run namespace), `corpus/<slug>/...` (per-brand research)
- **Enum key:** `UPPER_SNAKE` (CHANNEL, LANE, CLAIM_TYPE, BELIEF_KIND, EXECUTION_TYPE, PROOF_TIER, MOVE, etc.)

## Where to Add New Code

**New deterministic brick:**
- Location: `engine/bricks/<domain>-<verb>.js` (flat, no sub-dir)
- Require: self-documenting header (comment block with `@capability`, `@step`, `@inputs`, `@outputs`, `@run`, `@deps`, `@flags`)
- Dependencies: Shared lib goes in `engine/bricks/lib/` (e.g., `embed.js`); Python pixel work goes in `engine/bricks/asset/` (spawned from Node via explicit venv path)
- Pattern: Deterministic script (no LLM, no judgment); CLI args as `--flag=val` + positional; exit 0 (pass) or 1 (error) or 2 (validation reject); never fabricate missing data
- Test: Add smoke test to `engine/contracts/h6-<name>.sh` + fixture to `engine/_fixture/`; run `bash engine/contracts/h6-all.sh`

**New validator:**
- Location: `engine/hooks/validate-<producer>.js`
- Pattern: Accept filepath as argv[2]; import `../contracts/enums.json` for closed value sets; exit 0 (pass) / 2 (reject); stderr → issue messages
- Register: Add routing rule in `engine/hooks/route.js` (filename → validator)
- Test: Add test case to `engine/contracts/h6-firing.sh`

**New DR injector:**
- Location: `engine/hooks/inject-<consumer>-dr.js`
- Pattern: Hardcoded allowlist of DR files (security: no `../` traversal); configurable `--dr-dir=<path>` / `$DR_DIR` (wire-time override)
- Output: `--stdout` (orchestrator paste model) or `--out=<path>` (write file)
- Test: Add test case to `engine/contracts/h6-firing.sh`

**New record schema:**
- Location: `engine/contracts/schemas/<record>.schema.json`
- Pattern: Field shapes only (no semantic rules — those live in the corresponding validator)
- Reference: Enum keys from `enums.json` for value-set refs
- Status: After H0, all 6 schemas complete (brands, dump, space-map, funnel-fields, belief-record, asset-record)

**New enum / enum value:**
- Location: Add to `engine/contracts/enums.json`
- Pattern: Closed value set + `enforced_by` (list of validators) + `consumed_by` (list of consumers)
- Impact: Validators must import and check; HARD REJECT if off-enum
- Review: Must be approved by operator (value meanings live in operator's prompts; value sets live here)

**New integration:**
- Location: `engine/integrations/<service>/<service>-<verb>.{js,py,cjs}`
- Pattern: Use `lib-creds.js` for credential seam (--creds → env → homedir fallback); never hardcode secrets
- Dependencies: Isolated per service (no cross-service imports except `lib-creds.js`)
- Test: Dry-run to validate error handling (e.g., missing creds, API down); live test deferred to operator (R7)

**New agent prompt / SKILL:**
- Location: Outside `engine/` (operator/marketing owned); reference relevant brick/validator/injector by path
- Integration: Orchestrator calls validators explicitly (hooks don't fire in subagents); DR injectors spawned by orchestrator
- Pattern: Write SKILL.md in `.claude/skills/<name>/`; reference wiring via backtick paths

## Special Directories

**`engine/_fixture/`:**
- Purpose: Committed smoke-test inputs for h6-all.sh
- Generated: In-smoke (images/HTML/videos synthesized by test, no binary blobs)
- Committed: Fixture structure + golden outputs (e.g., `_tally.json`); binary-free

**`engine/bricks/asset/`:**
- Purpose: Python pixel-work bricks (probe, frame-grab, montage, video-assemble)
- Generated: None (source files only)
- Node-to-Python seam: Node bricks spawn Python via explicit `.venv/bin/python` (PEP 668 venv); pass argv arrays only (no string interpolation); read JSON from stdout
- bundled ffmpeg: imageio-ffmpeg (no system ffmpeg required)

**`engine/contracts/schemas/`:**
- Purpose: Record field-shape contracts
- Generated: Extracted from validators + live records (H0 hand-extraction, then never hand-maintained)
- Committed: All 6 schemas complete (H0, 2026-06-25)

**`engine/hooks/`:**
- Purpose: Firing layer (validators, router, injectors, firewall)
- Generated: Headers in bricks auto-generate REGISTRY.json (not hand-maintained)
- Registered: route.js is PostToolUse hook in `.claude/settings.json`; guard-marketing.js is PreToolUse (wired Phase 21, can be unwired)
- Subagent note: Hooks do NOT fire in subagents; orchestrator must call validators explicitly (see Section Analyzer, Asset-Classify patterns)
